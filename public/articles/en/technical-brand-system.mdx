# Designing a Technical Brand System Around One Portfolio URL

The portfolio should not compete with Medium, Velog, or community platforms. It should coordinate them.

That changes the role of `men16922.github.io`. It is no longer a resume-style landing page only. It becomes the system that explains identity, organizes evidence, and routes readers to the right depth.

## Channel roles

- Portfolio: brand hub and control plane
- GitHub Blog (MDX): primary technical publication surface
- Medium: global distribution
- Velog: Korean archive with lower publishing frequency
- Cloud communities: native platform publishing

## What this solves

Recruiters rarely inspect twenty links. They inspect one link and decide whether to continue. A single portfolio URL reduces lookup cost and increases the odds that projects, certifications, hackathons, and writing are evaluated together instead of as disconnected assets.

## Execution rule

Publish source content where it has the strongest audience fit, then always index it back under `/articles`.
