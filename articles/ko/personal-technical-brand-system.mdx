# 하나의 포트폴리오 URL 중심으로 기술 브랜드를 설계하는 방법

포트폴리오는 Medium, Velog, AWS Community를 대체하는 곳이 아닙니다. 이들을 통합하고 설명하는 중심축이어야 합니다.

따라서 `men16922.github.io` 의 역할은 단순 이력서 페이지가 아니라, 정체성 설명과 증거 정리를 담당하는 브랜드 허브가 됩니다.

## 채널 역할 분리

- Portfolio: 메인 브랜드 허브
- GitHub Blog (MDX): 메인 기술 블로그
- Medium: 글로벌 홍보 채널
- Velog: 한글 아카이브 채널
- 클라우드 커뮤니티: 플랫폼 네이티브 발행 채널

## 왜 이 구조가 유리한가

채용 담당자는 여러 플랫폼을 일일이 확인하지 않습니다. 하나의 URL에서 프로젝트, 자격증, 해커톤, 글, GitHub를 같이 보여주는 구조가 훨씬 평가 효율이 높습니다.

## 실행 원칙

어떤 채널에 글을 쓰더라도 최종적으로는 반드시 `/articles` 에서 다시 찾을 수 있어야 합니다.
