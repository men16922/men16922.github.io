# SlackOps Agent 아키텍처 정리: 알림을 운영 맥락으로 바꾸기

SlackOps는 단순히 Slack 안에서 동작하는 봇으로 보이면 약합니다. 더 적절한 설명은 알림과 엔지니어 사이에 위치한 운영 에이전트 시스템입니다.

## 문제

운영 알림은 양이 많고 맥락이 부족합니다. 실제 대응 전에 엔지니어가 상황을 재구성하는 비용이 큽니다.

## 시스템 목표

알림을 받아 장애 범위, 원인 후보, 다음 액션, 관련 증거 링크를 포함한 운영 브리프로 변환하는 것입니다.

## 포트폴리오에서의 설명 프레임

- Input: Slack 이벤트와 모니터링 알림
- Reasoning: agent workflow와 retrieval
- Execution: runbook, 이슈 생성, 알림 전파
- Evidence: 로그, 메트릭, 트레이스, 대시보드, 코드 상태

이 설명 방식은 AI를 도입했다는 사실보다, 운영 생산성을 어떻게 높였는지를 보여준다는 점에서 더 강합니다.
